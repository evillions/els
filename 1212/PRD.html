1.1 名词解释

名词 解释
卖家 发起众筹的音乐人
买家 参与并且购买了众筹相关产品的淘宝用户
专辑 卖家如果发布的是音乐众筹,且产出物包含一首(或多首)歌曲,称作专辑
封面 用于展示专辑内容的图片
单曲定价 众筹结束后,众筹产出物的数字音乐在虾米网下载的定价,非众筹项目内的定价
专辑标题 用于描述专辑的标题,可以与单曲标题相同
单曲演唱人 参与歌曲演唱的人员,比如合唱的歌曲就需要多个演唱人,可以与专辑归属人不一致
运营 负责对上传的专辑进行审核的人员

2 功能需求

用例 描述 优先级 备注
2.1 上传专辑 组件仅做数据展示,通过页面发布专辑 P0 通过发布专辑页面完成专辑上传,上传后进入后台审核,如果审核通过则进入音乐人专辑发布流程,专辑发布后7天可正常试听下载
2.2 上传内容后台审核 对通过上传组件上传的专辑进行审核 P0 审核通过的专辑才进行正常的音乐人发布专辑流程
2.2 用户下载 淘宝用户跳转至虾米网进行下载 P0 要实现淘宝虾米账号的绑定以及快速生成虾米账号的功能,http页面完成下载

2.1 上传专辑

  • 上传流程图

  • 描述

    上传专辑跳转至定制的淘星愿专辑发布页面进行,在发货弹窗内展示上传成功的专辑列表,页面完成上传后,专辑进入审核状态,需要提示专辑正在审核,通过后可在淘星愿后台进行发货.审核通过的专辑根据审核日期自动设置为7天后的上午10点发布(比如卖家是11月1日上传的专辑,发布时间就为11月8日上午10点),组件发布的专辑必须设置"仅虾币"下载

  • 规则

    1. 原有上传页面基本内容不变,取消定时发布按钮
    2. 原有的发布按钮文案调整为"提交审核"
    3. 专辑默认"仅虾币"
    4. 对卖家身份进行判断,非音乐人提示只有绑定虾米音乐人帐号才可以发布
    5. 发布时间默认为当天日期

  • 设计

    现有的专辑发布页面

  • 执行者

    卖家

  • 其他

    无

2.2 审核后台

  • 描述

    通过上传组件上传的专辑需要在后台进行审核,审核通过的专辑自动进入专辑发布流程

  • 规则

    1. 后台小编进行审核(24小时以内)
    2. 审核通过的专辑才进入正常发布流程
    3. 审核通过时通过接口把下载链接和审核状态传回众筹平台

  • 设计

  • 执行者

    运营

  • 其他

    无

2.3 用户下载

  • 描述

    买家在卖家发货后通过链接跳转到虾米页面完成数字MP3的下载流程

  • 规则

    1. 页面HTTP方式下载
    2. 淘宝账号必须绑定虾米账号才能完成下载
    3. 已绑定虾米账号的买家直接进行下载
    4. 每个用户只能下载一次(页面提示)
    5. 页面需要有明确的客服入口

  • 流程

  • 设计

    登录控件设计

  • 执行者

    买家

  • 其他

    无
